## Mocking Strategy

**All tests must mock every network call.** No test may ever hit a real vendor API.

The app makes four distinct fetch calls, all through `global.fetch`:

| Call | URL pattern | Made by |
|---|---|---|
| Oref alert history | `/api/history?mode=X&lang=X[&city=X][&fromDate=X&toDate=X]` | `lib/oref.ts → fetchAlertHistory` |
| Oref cities | `/api/cities?lang=X` | `lib/oref.ts → fetchCities` |
| Oref categories | `/api/categories` | `lib/oref.ts → fetchCategories` |
| TzevaAdom history | `/api/tzevaadom` | `lib/tzevaadom.ts → fetchTzevaadomRaw` |

**Standard mock pattern** for tests that need fetch:
```ts
beforeEach(() => {
  jest.spyOn(global, 'fetch').mockImplementation((url: RequestInfo) => {
    const u = String(url)
    if (u.includes('/api/history'))    return okJson(FIXTURE_ALERTS)
    if (u.includes('/api/cities'))     return okJson(FIXTURE_CITIES)
    if (u.includes('/api/categories')) return okJson(FIXTURE_CATEGORIES)
    if (u.includes('/api/tzevaadom'))  return okJson(FIXTURE_TZEVAADOM_RAW)
    return Promise.reject(new Error(`Unmocked URL: ${url}`))
  })
})
afterEach(() => jest.restoreAllMocks())
```

Reject unmocked URLs by default — this acts as a safety net to catch any new fetch calls that are added without a corresponding mock.

**Shared fixtures** should live in `tests/fixtures/`:
- `alertHistory.ts` — a small array of `AlarmHistoryItem` covering ≥2 cities, ≥2 categories, ≥2 days
- `cities.ts` — a short `City[]` array with both `label` (English) and `label_he` (Hebrew) fields
- `categories.ts` — `AlertCategory[]` covering `missilealert` and `uav` at minimum
- `tzevaadomRaw.ts` — a short `TzevaadomEntry[]` (`[rid, code, cities[], ts]`) covering ≥2 timestamps for date range testing

## 4. API Request Payload Snapshots

These snapshot tests lock in the **exact URLs and query parameters** sent to each API. They catch regressions like a parameter being dropped, renamed, or malformed during a refactor.

**How it works:** Mock `global.fetch` with `jest.spyOn`, then after triggering the relevant hook/function, snapshot `mockFetch.mock.calls` — the list of URLs actually requested.

**File:** `lib/__tests__/apiPayloads.test.ts`

### Oref alert history — `fetchAlertHistory`

| Snapshot name | Inputs | What it captures |
|---|---|---|
| `preset 24h, Hebrew, no city` | `mode=1, lang='he'` | `?mode=1&lang=he` |
| `preset 7d, Hebrew, no city` | `mode=2, lang='he'` | `?mode=2&lang=he` |
| `preset 30d, English, no city` | `mode=3, lang='en'` | `?mode=3&lang=en` |
| `preset 24h with city filter` | `mode=1, lang='he', city='תל אביב'` | `city` param included |
| `custom range` | `mode=0, fromDate='01.03.2026', toDate='12.03.2026'` | `fromDate` and `toDate` params present |

These ensure: mode values are not accidentally swapped, `city` is not double-encoded, date format sent to the API stays `DD.MM.YYYY`.

### Oref cities — `fetchCities`

| Snapshot name | Inputs | What it captures |
|---|---|---|
| `cities Hebrew` | `lang='he'` | `?lang=he` |
| `cities English` | `lang='en'` | `?lang=en` |

### Oref categories — `fetchCategories`

| Snapshot name | Inputs | What it captures |
|---|---|---|
| `categories` | none | URL has no extra params |

### TzevaAdom — `fetchTzevaadomRaw`

| Snapshot name | Inputs | What it captures |
|---|---|---|
| `tzevaadom raw` | none | `/api/tzevaadom` with no extra params |

**File:** `app/__tests__/page.test.tsx`

2. Missing lib unit tests (`tzevaadom`, `chartColors`, `filter` gaps) — fast, pure functions
3. **API payload snapshots** — high value, establishes the API contract baseline before any refactoring
4. `useCityRankings` hook tests — most complex logic in the app
5. `useAlerts` and `useTzevaadomAlerts` hook tests
6. `CityRankingChart` component tests — most logic-heavy component
7. `FilterBar` gaps + `LanguageToggle`
8. `ByDayChart` + `TimeOfDayChart` — basic smoke tests
9. Page-level user flow tests — broadest coverage, most setup
